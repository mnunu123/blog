## 한 줄 요약

> 제품명과 설명 몇 줄만 입력하면, AI 에이전트 5팀이 차례로 작업해 완성된 상세페이지 PNG/PDF를 뽑아준다.

---

## 문제

쇼핑몰 상세페이지 제작은 생각보다 복잡한 작업이다. 단순히 글을 쓰는 게 아니라:

1. **기획**: 타겟 고객과 페인포인트 분석
2. **카피라이팅**: Hero부터 CTA까지 13개 섹션별 문구
3. **디자인**: 색상, 폰트, 전체 톤 결정
4. **이미지**: 섹션별 시각적 레이아웃
5. **조합**: 전체를 하나의 파일로 합치기

이 과정을 한 명이 처음부터 끝까지 하면 반나절이 걸린다. 그리고 매번 비슷한 사고 흐름을 반복한다.

---

## 접근

### 왜 멀티에이전트인가?

단일 프롬프트로 "상세페이지 전체를 만들어줘"라고 하면 결과가 일관성 없고 길이가 제멋대로다. 각 역할을 분리하면:

- 각 에이전트가 **명확한 입력과 출력**을 가짐
- 에이전트별로 **다른 모델**을 써서 비용 최적화 가능
- 파이프라인 중간 결과를 **검토하거나 재시작**할 수 있음

### 파이프라인 설계

```
[입력] 제품 정보
    ↓
[01-intake   / claude-haiku]  필수 정보 수집 및 정제
    ↓
[02-research / claude-sonnet] 타겟/페인포인트 분석
    ↓
[03-copy     / claude-sonnet] 13섹션 카피라이팅
    ↓
[04-design   / claude-haiku]  스타일/컬러 프리셋 결정
    ↓
[05-prompt   / claude-sonnet] Gemini 이미지 프롬프트 생성
    ↓
[Gemini API] 섹션별 이미지 생성
    ↓
[스티칭] 이미지 합성 → PNG/PDF 출력
```

모델 선택 근거: 분석/카피 단계는 Sonnet(고품질), 정보 수집/디자인 결정처럼 단순한 단계는 Haiku(저비용).

## 구현

### 13개 섹션 구조

| # | 섹션 | 높이 | 목적 |
|---|------|------|------|
| 01 | Hero | 800px | 첫인상, CTA |
| 02 | Pain | 600px | 공감 유발 |
| 03 | Problem | 500px | 원인 제시 |
| 04 | Story | 700px | Before→After |
| 05 | Solution | 400px | 제품 소개 |
| 06 | How It Works | 600px | 프로세스 |
| 07 | Social Proof | 800px | 후기/증거 |
| 08 | Authority | 500px | 신뢰 구축 |
| 09 | Benefits | 700px | 혜택/보너스 |
| 10~13 | CTA, FAQ, Urgency, Footer | 가변 | 전환 유도 |

전체 높이 약 7,000px, 너비 1,200px 고정.

### 에이전트 디렉토리 구조

```
agents/
  01-intake/
  02-research/
  03-copy/
  04-design-direction/
  05-prompt-generator/
tools/               # 공통 유틸리티
scripts/             # 파이프라인 실행 스크립트
output/              # 생성된 PNG/PDF
```

각 에이전트는 독립적인 디렉토리를 가지며, 이전 에이전트의 출력 JSON을 입력으로 받는다.

---

## 시행착오

### 1. 에이전트 간 컨텍스트 전달

초기에는 에이전트 간 데이터를 메모리로 넘겼는데, 파이프라인이 중간에 실패하면 처음부터 다시 돌려야 했다. 각 에이전트가 **JSON 파일로 중간 산출물을 저장**하도록 바꿔서, 특정 단계만 재실행할 수 있게 했다.

### 2. Gemini 이미지 프롬프트 일관성

섹션마다 독립적으로 이미지를 생성하면 시각적 스타일이 들쭉날쭉했다. `04-design-direction` 에이전트가 결정한 **스타일 프리셋**(색상 팔레트, 분위기, 레이아웃 패턴)을 모든 이미지 프롬프트에 주입하는 방식으로 일관성을 확보했다.

### 3. 1,200px 고정 너비 + 섹션 높이 합산

섹션들을 수직으로 스티칭할 때 픽셀 단위 정밀도가 필요했다. 각 섹션의 높이를 config에 고정값으로 정의하고, 스티칭 스크립트에서 순서대로 합산하는 방식으로 안정화했다.

## 결과

- 제품 정보 입력 → 약 3~5분 내 `output/` 폴더에 PNG + PDF 생성
- 13섹션 완성형 상세페이지 자동 생성
- 섹션별 재생성(특정 에이전트만 재실행) 가능
