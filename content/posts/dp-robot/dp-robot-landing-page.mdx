## DP로봇이란

![DP로봇 — 자율주행 청소 로봇 MVP](/images/dp-robot/portfolio-p60.png)

DP로봇은 **자율주행 기반 청소 로봇** 프로젝트다. 핵심 기술 스택:

| 기술 | 상세 |
|------|------|
| **자율주행** | SLAM(동시 위치추정 및 지도작성) + A* 경로계획 알고리즘 |
| **객체 감지** | YOLOv5 기반 장애물 인식 |
| **배터리** | LiFePO4 (리튬인산철) — 안전성·수명 우선 |
| **제어** | ROS2 기반 모듈 통합 |

SLAM으로 실내 지도를 자동으로 생성하고, A* 알고리즘으로 최적 경로를 계산해 청소를 수행한다. YOLOv5가 실시간으로 장애물을 감지해 경로를 동적으로 회피한다.

![DP로봇 자율주행 구조 — SLAM + A* 경로계획](/images/dp-robot/portfolio-p61.png)

## 문제 정의

DP로봇은 물리적인 움직임이 핵심 가치인 키네틱 아트 로봇이다. 이걸 정적인 웹 페이지에서 어떻게 전달할 것인가.

동시에 실용적인 문제도 있었다. 디자인 수정이 생길 때마다 CSS 수치를 일일이 확인하고 코드로 옮기는 과정이 반복됐다. 미세한 레이아웃 조정이 많은 키네틱 스타일 페이지에서는 특히 시간이 낭비됐다.

두 문제를 동시에 해결하는 방식을 찾았다.

---

## 1. Figma MCP — 디자인이 코드로 직접 연결되는 구조

**MCP(Model Context Protocol)**: AI 에이전트가 외부 툴의 실시간 상태를 읽어올 수 있게 해주는 프로토콜. Figma API와 Cursor를 MCP 서버로 연결하면 Cursor가 현재 Figma 파일의 구조를 직접 읽는다.

기존 워크플로우:
```
Figma 수정 → 개발자가 수치 확인 → CSS 직접 수정
```

MCP 도입 후:
```
Figma 수정 → Cursor가 컨텍스트 읽기 → 코드 자동 업데이트
```

프레임을 단순 이미지가 아니라 React + Tailwind 컴포넌트 구조로 추출하도록 프롬프트를 설계했다.

```
"이 Figma 프레임을 Tailwind CSS 기반 React 컴포넌트로 변환해줘.
반응형 breakpoint는 sm/md/lg 기준으로 맞춰줘."
```

결과: 디자인 수정 → 코드 반영까지 직접 개입 없이 처리.

---

## 2. 키네틱 아트를 웹 애니메이션으로

로봇의 하드웨어 스펙을 나열하는 대신, **"움직임이 주는 경험"**에 집중한 레이아웃을 설계했다.

Framer Motion으로 로봇의 관절 움직임 메커니즘을 스크롤 애니메이션에 반영했다.

```tsx
// 스크롤 진행에 따라 회전·이동하는 요소
const { scrollYProgress } = useScroll()
const rotate = useTransform(scrollYProgress, [0, 1], [0, 360])
const x = useTransform(scrollYProgress, [0, 0.5], [0, 200])

<motion.div style={{ rotate, x }}>
  <RobotArm />
</motion.div>
```

Cursor에 요청한 방식:
```
"로봇 관절이 부드럽게 이동하는 느낌을 스크롤 애니메이션으로 구현해줘.
easing은 spring 기반으로, stiffness 낮게."
```

페이지를 스크롤하는 것만으로 로봇의 움직임을 간접 체험하는 구조.

---

## 3. 바이브 코딩 워크플로우

이번 프로젝트에서 실험한 개발 방식: 로직 구현은 Cursor에 맡기고, 나는 전체적인 분위기와 인터랙션 흐름을 결정하는 디렉팅에 집중.

실제 피드백 루프:
```
1. Figma에서 레이아웃 확인
2. Cursor에 MCP 컨텍스트로 수정 요청
3. 브라우저에서 즉시 확인
4. 수정 필요하면 1로 돌아감
```

이 루프를 최대한 빠르게 돌리는 게 핵심이었다. 코드를 직접 들여다보는 시간을 줄이고, "이 느낌이 맞는가"를 판단하는 시간을 늘렸다.

<Callout type="insight">
Cursor + MCP 조합에서 프롬프트의 품질이 결과물의 품질을 결정한다. "부드럽게"보다 "spring 기반, stiffness 80, damping 20"이 훨씬 정확한 결과를 뽑아낸다.
</Callout>

## 로봇 하드웨어 상세

랜딩페이지가 표현하려 했던 실제 로봇의 스펙:

![DP로봇 하드웨어 구성 상세](/images/dp-robot/portfolio-p63.png)

| 구성 | 사양 |
|------|------|
| **배터리** | LiFePO4 (리튬인산철) — 2000회 이상 충방전 사이클, 발화 위험 낮음 |
| **자율주행** | SLAM 기반 실내 지도 생성 + A* 최적 경로 계산 |
| **감지** | YOLOv5 실시간 장애물 인식 |
| **제어** | ROS2 노드 통합, 원격 모니터링 지원 |

LiFePO4 배터리를 선택한 이유는 청소 로봇 특성상 **장시간 운용**과 **안전성**이 우선이어서다. 일반 리튬이온 대비 에너지 밀도는 낮지만, 수명과 열 안정성에서 압도적이다.
